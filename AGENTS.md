# AGENTS.md - 행동 강령

## 핵심 원칙

1. **계획 먼저, 실행 다음**
   - 모든 개발 작업 전에 상세한 계획을 수립
   - todo list 로 작업 단위를 체크
   - 계획 승인 후 실행
   - 진행 상황을 실시간으로 표기

2. **모듈 단위 테스트 진행**
   - 하나의 모듈을 완전히 개발 후 다음으로 이동
   - 모듈 완료 후 반드시 테스트 수행
   - 실패하는 테스트가 있으면 해결 후 진행
   - 점진적으로 기능을 확장

3. **모든 테스트 완료 시 개발 완료**
   - 단위 테스트가 모두 통과해야 완료
   - 통합 테스트가 모두 통과해야 완료
   - 성능 테스트 (적용 시) 통과
   - LSP diagnostics 가 cleen 해야 완료

4. **모듈별 커밋 및 푸시**
   - 하나의 기능이 완료되면 즉시 커밋
   - 커밋 메세지는 명확하게 작성
   - 커밋 후 원격 저장소에 푸시
   - 히스토리 가 선명하게 남도록

## 개발 워크플로우

```
요청 받기
    ↓
전체 계획 수립 (todo list 생성)
    ↓
사용자 승인 (중요 작업 시)
    ↓
각 모듈에 대해:
    1. todo 상태 in_progress 로 변경
    2. 모듈 개발 수행
    3. 테스트 작성/수행
    4. pytest -v 실행
    5. 오류 해결
    6. todo 상태 completed 로 변경
    7. git add, commit
    8. git push
    ↓
모든 테스트 통과 (pytest -v)
    ↓
개발 완료
```

## 코드 품질 표준

- **타입 안전**: `as any`, `@ts-ignore` 사용 금지
- **오류 처리**: catch 문 비워두지 않음
- **LSP 검증**: 모든 변경 파일 diagnostics 확인
- **빌드 검증**: build 명령 통과 확인
- **테스트 커버리지**: 모든 기능에 테스트 포함

## Git 커밋 규칙

```
feat: 새로운 기능 추가
test: 테스트 추가 또는 수정
fix: 버그 수정
docs: 문서 수정
chore: 빌드 프로세스 또는 도구 수정
refactor: 코드 리팩토링
style: 코드 스타일 수정
```

## 작업 단계

### 개발 전
1. 요구사항 파악
2. todo list 작성
3. 의존성 식별
4. 기존 코드 패턴 확인

### 개발 중
1. 한 번에 하나의 모듈 실행
2. 기존 코드 스타일 유지
3. 테스트 병행 작성
4. lsp_diagnostics 검증

### 개발 후
1. 모든 todo 완료
2. 전체 테스트 통과
3. 빌드 성공
4. 커밋 및 푸시
5. 요청 내용 완전 반영

## 완료 확인 체크리스트

개발 완료 전 확인:

- [ ] 모든 todo 완료
- [ ] 모든 테스트 통과
- [ ] LSP diagnostics clean
- [ ] 빌드 성공
- [ ] 커밋 메시지 명확
- [ ] remote 에 푸시
- [ ] 사용자 요구사항 반영

## 실패 처리

1. 3 번 실패하면 바로 중지
2. 마지막 정상 상태로 롤백
3. 실패 사유 문서화
4. 사용자 또는 시니어 에이전트와 상담
5. 코드를 깨진 상태로 남기지 않음
